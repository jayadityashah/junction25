# EU Financial Document Comparison Frontend

A web application for viewing and comparing EU financial regulation documents, highlighting overlaps and contradictions.

## Structure

- `index.html` - Main HTML structure
- `styles.css` - Modern, responsive styling
- `app.js` - Application logic and data handling
- `analysis_results.json` - JSON file containing analysis results (to be generated by your RAG/LLM pipeline)

## Expected JSON Format

The `analysis_results.json` file should follow this structure:

```json
{
  "risk_categories": [
    {
      "name": "Credit Risk",
      "document_count": 12,
      "description": "Documents related to credit risk assessment"
    }
  ],
  "overlaps": [
    {
      "document1": "filename1.json",
      "document2": "filename2.json",
      "reason": "Why these documents overlap",
      "category1": "Basel",
      "category2": "CRD"
    }
  ],
  "contradictions": [
    {
      "document1": "filename1.json",
      "document2": "filename2.json",
      "reason": "How these documents contradict each other",
      "category1": "EBA",
      "category2": "FINLAND"
    }
  ]
}
```

## Running the Frontend

Simply open `index.html` in a web browser, or serve it with a local server:

```bash
# Python 3
python -m http.server 8000

# Then open http://localhost:8000 in your browser
```

## Integrating with SQLite Database

To load actual document content from your SQLite database, you'll need to:

1. Create a Python backend API (Flask/FastAPI) that queries the database
2. Update the `loadDocumentContent()` function in `app.js` to call your API
3. Or use sql.js to query SQLite directly in the browser
